# 勾留状謄本交付請求書 作成スキル

ユーザーが「勾留状謄本交付請求書を作って」「勾留状謄本交付請求書をコピーして本件に」等と指示した場合に発動する。

## 1. 入力の確定

ユーザーから以下を取得（指示文・添付パス・コンテキストから推定）：
- **依頼者**（例：田村正宣）
- **コピー元のひな形ファイルパス**（例：09_ワードファイル内の既存「勾留状謄本交付請求書.docx」）
- **本件の参照書類**（意見書・弁護人選任届等。事件名・裁判所判別の根拠）

不足する**本件固有の事実**（下記）は、AskUserQuestion で**まとめて最大3問**確認する：
1. 担当検察官名（内線含めば◎）
2. 勾留日（令和○年○月○日）
3. 在監場所（警察署名）

## 2. 田村案件等、依頼者固有情報の取得

被疑者の**生年月日・連絡先・関係者情報**は、依頼者フォルダ直下の **`<依頼者名>整理表.gsheet`**（例：`田村正宣整理表.gsheet`）の **「関係者連絡先等」シート**から取得する。

```python
# .gsheetからdoc_id取得 → Sheets APIで読み取り
import json, secretary
from googleapiclient.discovery import build

with open('<依頼者名>整理表.gsheet') as f:
    doc_id = json.load(f)['doc_id']

service = build('sheets', 'v4', credentials=secretary.get_credentials())
rows = service.spreadsheets().values().get(
    spreadsheetId=doc_id, range='関係者連絡先等!A1:Z50'
).execute().get('values', [])
# 被疑者氏名で行を探し、生年月日列（H列付近、'S44.10.18'形式）を取得
```

生年月日が「S44.10.18」形式なら → 「昭和４４年１０月１８日」に和暦変換。

## 3. 事件名・裁判所の判定

直近の意見書・弁護人選任届から：
- **事件名（罪名）**：「監禁、強要、強盗、窃盗被疑事件」等そのまま転記
- **宛先裁判所**：意見書宛先（東京地方裁判所等）と揃える
- **「再度」の有無**：
  - 初度の勾留（第1事件、または同一事件で初の勾留）→ **「再度」を入れない**
  - 再逮捕（第2事件・第3事件等）の勾留 → **「再度」を入れる**
- **弁護人選任届の発送日・発送先**：同フォルダ内の「弁護人選任届」docxから日付と検察庁名を取得

## 4. ファイル作成（別バージョン保存ルール）

**元ファイルを直接編集してはならない**（AGENTS.md/メモリの絶対ルール）。
- 元ファイル（卢范烨案件等の他案件流用）を `shutil.copy2` で同フォルダにコピー
- ファイル名末尾に `_2`（既に存在すれば `_3`、`_4`...）を付与
- 修正指示がさらに入ったら、その時点の最新版から **次の連番** にコピーして編集する

## 5. 本文の書き換え（python-docx）

段落番号は標準ひな形（260521勾留状謄本交付請求書.docx）基準：

| 段落 | 内容 |
|---|---|
| 2 | `担当検察官　○○　○○（内線　　　　　）` |
| 3 | `勾　留　日　○月　○日` |
| 4 | `在監場所　○○　警察署　` |
| 6 | `令和○年○月○日に○○地方検察庁に発送済` |
| 8 | `被疑者　　　　○○　○○（フリガナ）` |
| 9 | `生年月日　○○○○年　○月　○日　生`（和暦推奨） |
| 11 | `　上記の者に対する○○被疑事件につき，（再度）勾留状の執行を受けたので，その写しを交付されたく請求する。` |
| 13 | 作成日（令和○年○月○日） |
| 22 | 宛先裁判所（例：`東京地方裁判所　刑事部　御中`） |

**runを温存する置換ヘルパー**：
```python
def replace_paragraph(para, new_text):
    if para.runs:
        first_run = para.runs[0]
        for run in para.runs[1:]:
            run.text = ''
        first_run.text = new_text
    else:
        para.add_run(new_text)
```

## 6. 請書部分（テーブル1）も忘れず置換

ひな形末尾には**テーブル形式の「請書」**が含まれており、ここにも宛先裁判所（例：「盛岡簡易裁判所」）が**ハードコードされている**。本文の宛先を変えるなら**テーブル内の宛先も同じ裁判所に揃える**。

```python
for t in d.tables:
    for row in t.rows:
        for cell in row.cells:
            for para in cell.paragraphs:
                if '盛岡簡易裁判所' in para.text:  # 元ひな形の宛先
                    replace_paragraph(para, para.text.replace('盛岡簡易裁判所', '東京地方裁判所'))
```

## 7. 完了処理

1. `scripts/path_ops.py open` でファイルを開く（[[feedback_auto_open_documents]] 準拠）
2. ユーザーに**反映した値**と**空欄/要確認項目**（検察官の内線、不明な事実）を箇条書きで報告

## 注意点・落とし穴

- **段落のインデックスはひな形の改変で変わる**。書き換え前に `Document().paragraphs` を `print` で確認する
- **`paragraph.text = ...` を使うとrunが壊れて書式が崩れる**。必ず上記`replace_paragraph`でrun温存
- **「再度」の有無は事件の経緯から判定**。ユーザー指示で「再度ではない」と訂正されたら今後の同種案件で同じ間違いをしない
- **生年月日を「不明」のまま提出しない**。整理表に必ず当たる
- 日本語パスを Bash 直書きしない（AGENTS.md：`.cursor/shell_path_utf8.txt` 経由）

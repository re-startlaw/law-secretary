## A. Codex運用チップス（米谷先生がまだ未着手の3つ）

メモリ・現状の使い方を踏まえて、**作るのが簡単で効果が出やすいもの**を3つ選びました。順番は導入のしやすさ順です。

---

### A-1. プロジェクト固有のskill（`.agents/skills/`）

**何ができるか**
事務所で繰り返し依頼している処理（例：請求書作成、ファイル分類、契約書ドラフト）を、`.agents/skills/<名前>/SKILL.md` に「いつもの手順」として置けます。Codexは依頼内容がskillの説明に合うと自動で読み込み、明示的に `$skill-name` と指定して呼ぶこともできます。

**作り方（最小例）**

```bash
mkdir -p .agents/skills/invoice
```

`/Users/kometaninaoki/law-secretary/.agents/skills/invoice/SKILL.md` に以下のような中身を置く：

```markdown
---
name: invoice
description: 請求書作成ワークフロー（メール抽出→初回相談料控除判定→freee作成）
---

`docs/task_invoice_accounting.md` の手順に沿って、請求書作成を支援する。
- 事業区分: 法人（既定）
- 支払期日: 月末（既定）
- 件名: 委任契約書の事件名と揃える
- 金額・期日・送信先メールは実行前に必ず復唱確認する
- 完了したら freee の請求書URLを報告する
```

これでCodexに「請求書を作成して」または「$invoice で請求書を作成して」と依頼すれば、プロンプトをゼロから書き起こさず同じ手順に寄せられます。プロジェクトスコープ（`.agents/skills/`）に置けばgitで共有できます。

**米谷先生にとって特に効くポイント**
- 「請求書作成」「ファイル分類」「連絡書面作成」など、すでに固まっている定型ワークフローを **skill 1個** に閉じ込められる
- ルールファイル（`docs/task_*.md`）を毎回手で参照させず、skill側に「これを読め」と書ける
- 事務員（佐藤さん）が同じ環境を使う場合も、git push しておけば共有される

**実務インパクト**：★★★ / **導入コスト**：★（5分）

[参考: Codex Agent Skills](https://developers.openai.com/codex/skills)

### A-2. Codexのsubagentワークフロー＋専門skill

**何ができるか**
Codexでは、ユーザーが明示的に「subagentで」「並列エージェントで」と頼むと、調査・レビュー・ログ分析などの読み取り中心タスクを分担できます。恒常的な専門手順はskillにしておき、必要なときだけsubagentに「このskill相当の観点で見て」と渡すのが扱いやすいです。

**米谷先生に効きそうな子エージェント案**

| 名前 | 役割 | ツール許可 |
|---|---|---|
| `bunsho-kosei`（書面校正） | 民事準備書面・意見書を読み、誤字・敬語・条文番号の整合・「グレーアウト処理」が漏れていないかチェック | ファイル閲覧・検索のみ |
| `hanrei-research`（判例リサーチ） | 与えられた論点で westlaw / 裁判所HP / 公開判例を検索し短報 | 利用可能な外部検索・ブラウザ |
| `keisan-checker`（請求金額チェック） | 着手金・実費・控除（初回相談料）の計算とfreee入力前の整合確認 | ファイル閲覧・必要最小限の計算コマンド |

**作り方（最小例）**

まずは `.agents/skills/bunsho-kosei/SKILL.md` として専門観点を保存する：

```markdown
---
name: bunsho-kosei
description: 民事準備書面・意見書の校正専門。誤字・敬語・条文番号・グレーアウトをチェック
---

あなたは法律書面の校正者です。以下を順にチェックしてください：
1. 誤字・脱字・てにをは
2. 敬語の統一（です・ます調 / である調）
3. 引用条文番号と正文の整合
4. 「本件非該当記述はグレーアウト」が抜けていないか
5. 当事者名・事件番号の表記揺れ

修正案は箇条書きで返す。直接ファイルは編集しない。
```

メイン会話側では「準備書面.docxを `$bunsho-kosei` の観点でチェックして」と頼む。複数観点を同時に見たいときだけ、「書面校正・証拠対応・金額計算をsubagentで並列に確認して」と明示する。

**実務インパクト**：★★★ / **導入コスト**：★★（30分、慣れが必要）

[参考: Codex Subagents](https://developers.openai.com/codex/concepts/subagents)

### A-3. 個人用custom prompt（補助的に使う）

**何ができるか**
Codexでは、共有したい反復手順はskillが本命です。一方、自分だけが使う短い定型依頼は `~/.codex/prompts/*.md` に置いて、`/prompts:<名前>` として呼び出せます。custom promptは明示呼び出し専用で、リポジトリ共有には向きません。

**なぜ未着手で効くか**
毎回少しだけ違う依頼（例：今日の予定整理、短いレビュー依頼、PR説明文作成）はskill化するほどではないため、個人用custom promptにすると速いです。ただしCodex公式ではcustom promptよりskillが推奨されているため、業務ワークフローはskillへ寄せます。

**最小例**

`~/.codex/prompts/review-doc.md`：

```markdown
---
description: 法律書面の短時間レビュー
argument-hint: FILE= FOCUS=
---

$FILE を確認し、$FOCUS を中心に、重大な誤り・確認事項・修正提案だけを簡潔に列挙してください。
直接編集はしないでください。
```

**実務インパクト**：★★ / **導入コスト**：★（10分）

[参考: Codex Custom Prompts](https://developers.openai.com/codex/custom-prompts)

## B. 刑事弁護ニュース（2026年4月、制度2件＋個別事件1件）

### B-1. 再審法改正案、検察抗告「原則禁止」で再修正・連休明けに先送り（制度／最重要）

**要点**
- 政府は再審法改正案について、再審開始決定への検察抗告を **「原則禁止」** と明記し、「決定を取り消すべき十分な理由」がある場合のみ例外、とする再修正案を準備中。
- 5年後ではなく **「5年ごと」** の見直し規定を付則に格上げ。
- 再審請求の早期棄却要件の一部を削除する方向。
- 4月10日提出予定だったが自民党内で異論続出（稲田朋美氏らが「修羅場」化）し、5月連休明けに先送り確定。
- 一方で **「開示証拠の目的外使用禁止」** 規定が残る点について、5月8日に日弁連がシンポジウムを開催予定（弁護士会館クレオA、Zoom配信あり）。

**実務的に押さえておくべき点**
- 通常事件では「目的外使用禁止」が当たり前ですが、再審請求審で同じ規定を入れると **支援者・市民への証拠共有が萎縮**しかねず、袴田事件の弁護団が強く反対している論点。
- 「原則禁止」は条文化されても **例外要件「十分な理由」が広く運用される懸念**（古舘伊知郎氏・伊藤和子弁護士らが指摘）。
- 国会提出時期によっては、刑事弁護研修・所内勉強会のテーマに直結。

**要点**
- 2026年4月6日、大川原化工機元顧問・相嶋静夫さん（勾留中に胃がんで死亡）の遺族が、**保釈請求却下や逮捕状・勾留状発付に関与した裁判官37人を実名で挙げ**、国に対し約1.69億円の損害賠償を求める訴訟を東京地裁に提起。

### B-3. 日本版司法取引、警察送致事件・地方自治体事案へ拡大（制度／実務動向）

**要点**
- 2024年〜2025年にかけて **東京地検特捜部以外** が手がけた事案で日本版司法取引が複数適用されたことが判明（西村あさひ N&A ニューズレター 2026/01/30 が整理）。
  - 第4号: 警察捜査初の司法取引（四国銀行詐取・税理士、神戸地判令和6/10/17）
  - 第5号: 大阪地検特捜部・奈良県御所市火葬場贈収賄事件
  - 第6号: 群馬・桐生官製談合事件（さいたま地判令和7/11/25）
- 2025年10月、**最高検が特殊詐欺等の組織的犯罪に積極適用する方針**を公表。最高検・各高検に司法取引運用を指導する検事を新たに配置し、全国レベルで統一運用を可能にする体制構築へ。

**実務的に押さえておくべき点**
- これまで「司法取引は東京地検特捜部の独自捜査事件のみ」という理解でほぼ足りていたが、**警察送致事件・地方の贈収賄・官製談合・特殊詐欺**にまで広がる前提に切り替え必要。
- 共犯者がいる否認事件の弁護方針検討段階で、「相手側に司法取引を提案される可能性」を必ず織り込む。
- 弁護人として被疑者と協議する場合、合意後の供述の信用性争い・反対尋問の準備が次のフェーズ。
